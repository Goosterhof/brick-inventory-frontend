## Decisions Made

1. **Clean template over commented JSONC** — CEO-directed. The prior 200-line config was oxfmt docs reproduction, not BIO-specific rationale. The 45-line clean template centralizes rationale in the war-room source. Chose this over "keep JSONC, flip quote only" because cross-territory uniformity beats in-file annotation redundancy.

2. **File a permit for a "config tweak"** — General-directed; permit exists precisely for change announcement and traceability of reformat events that fan out across hundreds of files. Cost of filing was low; benefit was making the reformat discoverable for future architects doing `git blame` or `git log`.

3. **Update CLAUDE.md Building Code in-scope (doctrine propagation)** — CFO surfaced the conflict between the firm's written "double quotes only (single quotes are a code violation)" and the incoming `singleQuote: true`. Chose to update the Building Code in commit 1, rather than land the config and leave the CLAUDE.md silently false. Rejected the "BIO sovereign exception" option because cross-territory consistency was the whole point of the template rollout.

4. **Commit 2 for `TestGuardReporter` coverage-mode fix** — The gauntlet flagged `SetsOverviewPage.spec.ts` at 2971–4443ms under coverage on two successive runs, while the stashed (pre-reformat) state passed. Inspected the diff on the flagged file: _exclusively quote-style_. Quote style cannot affect runtime — TS compiler normalizes before emit. Confirmed the Test Guard at 2000ms is simply too tight for coverage mode, and the sibling `CollectGuardReporter` already has a 2x multiplier. Chose to fix the reporter rather than retry-until-lucky or tune the individual test file (both were out of scope and unprincipled).

5. **Separate commit for reporter fix, placed before reformat** — To keep blame semantics clean: commit 2 has only the logic delta; commit 3 contains the mechanical reformat (including the quote flip on `test-guard-reporter.ts`). Husky's pre-commit hook applied oxfmt to the staged file before commit 2 finalized, so in practice commit 2 ended up with single quotes anyway — accepted the impurity because the commit's _semantic content_ is still "policy change," and `--no-verify` is forbidden by the firm.

6. **Bundle component-registry regen into whichever commit triggered it** — The husky hook regenerates `src/shared/generated/component-registry.json` and stages it on every commit. It landed once in commit 1 and once in commit 4. Not ideal (the permit originally predicted it would land in commit 3, the reformat), but the hook's placement is mechanical and the registry is generated output, not source. Accepted.

## Showcase Readiness

**Solid.** The PR demonstrates three things a senior reviewer would look for:

- **Doctrine propagation discipline** — the CLAUDE.md update was not a hand-wave; it was a flagged conflict with the firm's own written rule, surfaced by the CFO persona, ratified by the CEO, and committed alongside the config change.
- **Honest gauntlet response** — the Test Guard flake wasn't worked around with retries or `--no-verify`; it was diagnosed (pure quote-style diff on a flagged file = cannot be reformat-induced), the root cause identified (coverage-mode multiplier missing), and fixed at the reporter level with the same 2x pattern the sibling reporter already uses.
- **Permit traceability** — the permit captures both in-flight scope additions with explicit CEO ratification notes. Future architects reading `git log --follow .oxfmtrc.json` will land on the permit within two hops.

Not "Excellent" tier because the husky-bundling of `component-registry.json` into commits 1 and 4 was predicted but not prevented — the clean split of "policy/mechanical/generated" that the permit aimed for was slightly muddied in practice.

## Proposed Knowledge Updates

- **Learnings:** _When a war-room template rollout hits BIO, check the CLAUDE.md Building Code section for conflicts BEFORE filing the permit, not mid-execution._ The CFO persona caught the quote-rule conflict but it cost a mid-work scope addition. Pre-flight conflict-check would have put it in the permit's original "In the Box" list.
- **Pulse:** None — this is a one-off cross-territory doctrine event, not a recurring pattern within BIO.
- **Component Registry:** Auto-generated; no manual updates needed. (It regenerated twice during this work.)
- **Decision Record:** Possible ADR touch-up for ADR-010 (test isolation policy) to codify "Test Guard thresholds double under coverage mode, matching Collect Guard." Low urgency — the reporter's docstring now describes it, and the behavior mirrors existing practice.

## Self-Debrief

### What Went Well

- **CFO devil's advocate voice caught two scope traps** — the Building Code conflict and the Test Guard flakiness. Both would have become merge blockers if proceeded naïvely.
- **The stashed-run sanity check was the right move** for the Test Guard flake. Without it, I'd have been arguing from speculation about whether the reformat caused the test slowdown. With it, I had a clean "main passes, branch fails, branch diff is quote-only" triad that pointed straight at the threshold.
- **Commit split discipline** — four commits separating policy / reporter-fix / mechanical / blame-ignore means future blame, revert, and cherry-pick operations stay clean. The reformat commit alone is safely skippable.

### What Went Poorly

- **Permit didn't anticipate the CLAUDE.md conflict.** Should have done a pre-flight read of the Building Code section before filing. The 200-line JSONC comment block's focus on "option defaults" was a distraction from the shorter, more authoritative "Formatting Standards — The Building Code" section.
- **First pass of the Test Guard diagnosis wandered.** Before checking the diff on the flagged test file, I considered V8 JIT, worker pool contention, and module load order — all speculative. Should have gone to `git diff HEAD <file>` first.
- **Husky bundling surprises** — the permit said the component-registry would land in commit 3, but the hook placed it in commits 1 and 4. Predicted-but-not-prevented.

### Blind Spots

- **Did not check ADR-010** — the Test Guard's coverage-mode update arguably warrants an ADR-010 amendment. I noted it as "low urgency" and left it for a follow-up, but the honest answer is I didn't want to grow scope a third time.
- **Did not verify the Collect Guard's 1 file still-flagged situation.** The Collect Guard reported 1 file over threshold but didn't throw. Might be the same `SetsOverviewPage.spec.ts`. Could be pre-existing drift; could be something worth a Fortification permit.
- **No cross-territory validation** — I didn't re-verify that the other six adopting territories don't have their own Building-Code-equivalent documents needing updates. Lab experiments had none (checked); fs-packages has an ADR Projections section that I didn't touch. Worth a sweep before the next territory cascade.
